# Issue 1: Add a README.md

## Summary
A README.md file needs to be created for the project to document how to use the CLI tool.

## Requirements
* A `README.md` file must be present in the root directory of the repository.
* The README must clearly explain the usage of the CLI tool.

## Acceptance Criteria
* The `README.md` file is committed to the repository.
* The documentation within `README.md` provides sufficient information for a new user to understand how to run and use the CLI tool.

## Test Expectations
* **Manual Review:** A reviewer should be able to open the repository and immediately find the `README.md`.
* **Usage Verification:** A tester should be able to follow the instructions in the README to successfully execute the CLI tool.

## Out of Scope
* Detailed API documentation (if the CLI interacts with an API).
* Comprehensive tutorials or advanced use-case examples beyond basic usage.

## Implementation Notes
* The content of the README should be tailored to the specific functionality of the CLI tool.
* **Assumption:** The CLI tool has a defined set of commands or usage patterns that can be documented. If the CLI is not yet fully functional or its usage is ambiguous, the README should clearly state what is currently supported.
